# Solnew

A modern, self-hosted **cryptocurrency faucet** built on the lightweight Fernico
PHP framework with Smarty templates. Originally based on the Volcrado faucet
script - this revision rebuilds the codebase from the ground up: every known
bug is fixed, all SQL is parameterised, CSRF is enforced everywhere, and the
front-end has been replaced by a brand-new theme called **Nova**.

---

## What's new in this rebuild

### Bugs fixed

- **Missing entry point** - the original repo had no root `index.php`, so the
  framework couldn't boot. Added `/index.php` and `/lib/Astrid/init.php`.
- **`fernico_desotry()` typos** - three calls in `accountController.php` were
  spelled wrong and would have triggered fatal errors. All replaced with
  `fernico_destroy()`.
- **SQL injection** - the admin and faucet flows interpolated user input
  (and `$_SESSION['user_id']`) directly into SQL. Every query is now a
  prepared statement.
- **`affected_row` / `affected_rows`** - typo in the email-change flow
  prevented the INSERT-when-no-row-exists fallback from firing.
- **Installer column-name bug** - `UPDATE config SET value = ? WHERE name = ?`
  used the wrong column name; the actual column is `parameter`.
- **Installer rmdir case mismatch** - `rmdir(..."installer")` couldn't find
  the directory which is actually `Installer`.
- **`fernico_getAbsURL()` scheme detection** - the original logic concluded
  that any port other than 80 was HTTPS. Now uses
  `HTTP_X_FORWARDED_PROTO` / `HTTPS` / `SERVER_PORT` in that order.
- **Dead `freegeoip.net` API** - replaced with `ipinfo.io`.
- **Dead FaucetHub.io check** during registration removed (the service shut
  down in 2020).
- **PHP 7.1+ session deprecations** - removed `session.entropy_*` and
  `session.hash_*` ini settings; uses `SameSite=Lax` cookies and periodic
  `session_regenerate_id()` instead.
- **Smarty `sysplugins/` path mismatch** - the vendored copy of Smarty puts
  its sysplugins at `/lib/sysplugins/` rather than `/lib/Smarty/sysplugins/`.
  `DependenciesLoader.php` now sets `SMARTY_SYSPLUGINS_DIR` before loading
  Smarty so the include resolves either way.
- **Empty `config/config.php`** - replaced with a `config.example.php` that
  the installer copies into place.

### Security hardened

- **CSRF tokens** are generated by the framework and required on every
  state-changing POST (login, register, password change, address change,
  admin actions, ads, settings).
- **All SQL is parameterised** with `mysqli` prepared statements.
- **Templates escape output** with `{$var|escape:'html'}` everywhere.
- **Admin password storage** migrated to `password_hash()` with a
  back-compat path for legacy SHA-512 hashes.
- **Cookies** opt in to `Secure` / `HttpOnly` / `SameSite=Lax`.
- **Security headers** (`X-Content-Type-Options`, `X-Frame-Options`,
  `Referrer-Policy`, `Permissions-Policy`) sent from `.htaccess`.
- **Session ID rotation** every 30 minutes to mitigate fixation.

### New theme: Nova

- **Dark-first design** with a one-click light-theme toggle (preference is
  remembered in `localStorage` and respects `prefers-color-scheme`).
- **Zero CSS / JS framework dependencies** - one `nova.css` (~600 lines) and
  one `nova.js` (~150 lines, vanilla, no jQuery).
- **System font stack** - no external font loading.
- **Inline SVG icons** instead of Font Awesome.
- **Responsive** mobile-first layout using CSS Grid + Flexbox.
- **Accessible** skip-link, keyboard-friendly nav, `prefers-reduced-motion`
  honoured.
- **No anti-adblock script** - the obfuscated payload from the original
  footer was removed.

## Requirements

- PHP **7.1+** (8.0 / 8.1 / 8.2 tested)
- MySQL / MariaDB with **mysqlnd**
- Apache with `mod_rewrite` (Nginx works too with an equivalent rewrite)
- PHP extensions: `mbstring`, `cURL`, `mysqli`, `random_bytes`, `password_hash`
  (the installer pre-flight checks for these)

## Installation

1. Upload the project to your webroot (or `git clone` it).
2. Make sure `/config/`, `/resources/`, and `/storage/` are writable by
   PHP.
3. Visit your site in a browser; if `config/config.php` doesn't yet exist
   the framework hands off to the installer at `/resources/Installer/`.
4. Fill in the form (database details + admin user) and press **Install**.
5. The installer creates the schema, seeds the `config` table, writes
   `config/config.php`, and removes itself. You're redirected to the admin
   sign-in page.

## Configuration

`config/config.example.php` documents every available key. The interesting
ones you'll likely change:

| Key                 | Purpose                                       |
|---------------------|-----------------------------------------------|
| `WEBSITE_URL`       | Force the public URL (with trailing slash).   |
| `COOKIE_SECRET`     | Random secret used for the remember-me hash.  |
| `SECURE`            | Set `true` once you serve over HTTPS.         |
| `TEMPLATE_DIR`      | Active theme name. Defaults to `Nova`.        |
| `SESSION_DAYS`      | Days before a session expires.                |

Site-level settings (faucet reward, captcha keys, FaucetPay key, SMTP
credentials, etc.) live in the `config` table and are editable via
`/admin/settings`.

## URL routing

The router maps URLs of the form `/<controller>/<action>/<param1>/<param2>`
straight to `controllers/<controller>Controller.php::<action>(<params>)`.
Hyphens in URLs translate to a double-underscore in PHP, e.g.
`/account/reset-password` -> `accountController::reset__password()`.

| URL                              | Controller        | Notes              |
|----------------------------------|-------------------|--------------------|
| `/`                              | homeIndex::home   | Landing page       |
| `/account/login`                 | account::login    |                    |
| `/account/register`              | account::register |                    |
| `/account/reset-password`        | account::reset__password |             |
| `/account/settings`              | account::settings | Logged-in only     |
| `/page/dashboard`                | page::dashboard   | The faucet         |
| `/page/affiliate-programme`      | page::affiliate__programme | |
| `/admin/login`                   | admin::login      | Cookie-auth area   |
| `/admin/home`                    | admin::home       |                    |

## Faucet flow

1. User signs in and clicks **Claim my reward** on `/page/dashboard`.
2. Captcha is verified server-side, a one-shot `claims_hashes` row is
   created with a random 64-char hash, and the user is redirected through
   the configured shortlink (ouo.io / shorte.st) to
   `/page/dashboard/<hash>`.
3. When the user returns, the hash is looked up, FaucetPay is called, the
   referral payout (if any) fires, and the hash is removed from
   `claims_hashes`.
4. A row is recorded in `claims_registered` for the audit log on the home
   page.

## Theming

The active theme directory name is stored in `TEMPLATE_DIR`. To create
your own theme, copy `views/Nova/` and `resources/Nova/` to a new name,
then point `TEMPLATE_DIR` at it. Each template is a Smarty 3 `.tpl` file
and may pull static assets from `/resources/<theme>/...` via
`{App::makeLink('css/yours.css', true)}`.

## Theme switcher

The Nova theme ships with a sun/moon button in the navbar. Clicking it
flips the `data-theme` attribute on `<html>` between `dark` (default) and
`light`. The choice is persisted in `localStorage` under `nova-theme`.
